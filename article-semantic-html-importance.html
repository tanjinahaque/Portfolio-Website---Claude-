The Importance of Semantic HTML

Semantic HTML gives browsers and assistive technologies a clear map of the page. It is one of the simplest ways to make an experience more durable, understandable, and easier to test.

Native Elements Carry Meaning

A real button communicates its role, supports keyboard activation, and exposes its state in a predictable way. A clickable div has to recreate that behavior with JavaScript, ARIA, focus management, and key handling.

  • Use <button> for actions like opening a modal, submitting a choice, or expanding content.
  • Use <a> with a real href for navigation to another page, file, or location.
  • Use lists, tables, headings, and form controls when the content actually has that structure.

Real Example: Card Links

If a portfolio card opens a case study, the card should contain a clear link with meaningful text, such as "State Benefits Portal Audit case study." The visual card can be made fully clickable, but the accessible name should still explain the destination.

Landmarks Help Users Navigate

Using <header>, <nav>, <main>, and <footer> gives screen reader users reliable regions to jump between. This is especially helpful on long pages with repeated content.
