Accessibility Fundamentals

Understanding Severity, User Impact, and Priority

A practical reference for helping product teams understand what accessibility severity means, how user impact should shape decisions, and why priority is more than a checklist score.

Severity Describes the Barrier

Severity should explain how much an issue blocks or disrupts a user. A severe issue is not just a failed requirement; it is a barrier that can prevent someone from completing a task, understanding information, or recovering from an error.

  • Critical issues block core tasks or create complete loss of access.
  • High-severity issues create major friction, confusion, or risk in important workflows.
  • Lower-severity issues still matter, especially when they repeat across components or templates.

User Impact Gives the Issue Meaning

User impact connects the defect to the person affected. This helps product owners understand who is blocked, what assistive technology or interaction pattern is involved, and what the issue means in the context of the workflow.

Priority Balances Risk, Timing, and Ownership

Priority is the decision about when the work should happen. It should consider severity, user impact, frequency, release timing, technical effort, and whether the same pattern creates repeated issues elsewhere.
