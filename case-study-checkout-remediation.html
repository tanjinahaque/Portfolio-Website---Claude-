Accessibility Case Study

Checkout Flow Remediation

A focused accessibility remediation effort for a high-traffic e-commerce checkout where keyboard traps and focus failures were preventing customers from completing purchases.

Role
Accessibility Analyst
Timeline
2 Sprints
Scope
Checkout Flow
Standards
WCAG 2.1 AA

Overview

The checkout flow served roughly 40K monthly users and included cart review, shipping, payment, promotions, account sign-in, and confirmation steps. I audited the flow, isolated blockers, and partnered with developers to validate remediation across each checkout step.

Challenge

The most urgent barrier was a keyboard trap in the multi-step checkout, but the larger issue was inconsistent focus behavior across overlays, validation messages, and step transitions.

  • Keyboard users could not reliably exit promotional and payment overlays.
  • Focus sometimes moved behind modals or returned to the wrong step.
  • Error messages were visually clear but not consistently associated with fields.
  • Progress updates were not always announced after step changes.

Process

  • Reproduced the keyboard trap and documented the exact tab sequence that caused it.
  • Mapped expected focus behavior for each checkout step, modal, validation event, and confirmation state.
  • Reviewed ARIA attributes, field labels, live regions, and error associations.
  • Created regression checks for keyboard-only, screen reader, zoom, and mobile viewport testing.
  • Validated fixes with developers before release and again after deployment.

Key Findings

  • Modal focus trapping was incomplete and did not restore focus to the triggering control.
  • Several disabled-looking controls remained keyboard-focusable.
  • Payment field errors needed stronger programmatic relationships and clearer recovery instructions.
  • Step changes needed consistent headings, focus targets, and status announcements.

Impact

  • 6 checkout steps verified with keyboard trap remediation.
  • 100% focus management coverage for audited modal and step-change patterns.
  • 8% conversion uplift after checkout accessibility fixes shipped.

Reflection

This project reinforced how accessibility defects can become business defects too. Fixing keyboard and focus behavior did not only help customers using assistive technology; it made the checkout flow more predictable for everyone.
